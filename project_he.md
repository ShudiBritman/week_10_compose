בהשלמת פרויקט זה תדגים:

## סקירת הפרויקט

תבנה **API לניהול אנשי קשר** – שירות REST המאפשר ניהול רשימת אנשי קשר. האפליקציה:

- שומרת אנשי קשר במסד נתונים MySQL  
- מספקת נקודות קצה HTTP לפעולות CRUD (יצירה, קריאה, עדכון, מחיקה)  
- רצה בקונטיינרים של Docker באמצעות Docker Compose  
- שומרת נתונים מתמידים באמצעות Docker Volumes

**טכנולוגיות:**

- Python 3.11+  
- FastAPI  
- MySQL 8.0  
- Docker & Docker Compose

**משך:** יומיים

## דרישות פונקציונליות

### 1. מודל נתונים

כל איש קשר חייב לכלול את השדות הבאים:

| שדה | סוג | אילוצים |
| --- | --- | --- |
| id | מספר שלם | מפתח ראשי, Auto‑increment |
| first_name | מחרוזת (50) | לא ריק |
| last_name | מחרוזת (50) | לא ריק |
| phone_number | מחרוזת (20) | לא ריק, ייחודי |

### 2. נקודות קצה API

ה‑API חייב לממש 4 נקודות קצה:

| שיטה | נתיב | תיאור | גוף בקשה | תגובה |
| --- | --- | --- | --- | --- |
| GET | `/contacts` | קבלת כל אנשי הקשר | אין | רשימת אנשי קשר |
| POST | `/contacts` | יצירת איש קשר חדש | נתוני איש קשר | הודעת הצלחה + מזהה |
| PUT | `/contacts/{id}` | עדכון איש קשר | שדות מעודכנים | הודעת הצלחה |
| DELETE | `/contacts/{id}` | מחיקת איש קשר | אין | הודעת הצלחה |

**דוגמה:**

POST /contacts

```json
{
  "first_name": "John",
  "last_name": "Doe",
  "phone_number": "050-1234567"
}
```

**תגובה:**

```json
{
  "message": "Contact created successfully",
  "id": 4
}
```

### 3. פעולות מסד נתונים

יש לממש ללא ORM:

- יצירת איש קשר (INSERT)  
- שליפת כל אנשי הקשר (SELECT)  
- עדכון איש קשר (UPDATE)  
- מחיקת איש קשר (DELETE)
